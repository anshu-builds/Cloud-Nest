CloudNest

Frameworks, Languages & Tools Report

Distributed Cloud File Storage System · Java 21 / Spring Boot 3.4.5 · May 2026

1. Project Overview

CloudNest is a simplified distributed cloud file storage system — a "Mini Google Drive" — built as a Master's-level university project. It demonstrates enterprise architecture patterns including user authentication, hierarchical file/folder management, file sharing via unique links, and a dashboard with storage usage analytics and distributed node distribution metrics.

Key Capabilities: User registration & login · File upload/download/delete (max 50 MB) · Nested folder management · Shareable public links with 7-day expiry · Dashboard with statistics & node distribution chart · Simulated 3-node distributed storage.

3. Layered Architecture

CloudNest follows the classic Spring MVC layered architecture pattern:

Browser → Controller → Service → Repository → MySQL ↓ Thymeleaf Templates → HTML Response → Browser

Each layer has a single responsibility: Controllers handle HTTP requests, Services contain business logic, and Repositories execute database queries via JPA.

4. Detailed Framework Descriptions

4.1 — Spring Boot 3.4.5

Spring Boot v3.4.5

What it is: An opinionated framework built on top of the Spring Framework that simplifies the creation of production-ready Java applications. It provides auto-configuration, an embedded web server, and starter dependencies.

How we use it: Spring Boot is the backbone of CloudNest. The main class CloudNestApplication.java bootstraps the entire application with a single @SpringBootApplication annotation. It auto-configures the embedded Tomcat server, database connections, security filters, and Thymeleaf template resolution — eliminating hundreds of lines of XML configuration.

What it does in the project: Provides dependency injection via @Autowired, manages the application lifecycle, bundles the app as an executable JAR, and enables hot-reload during development via DevTools.

4.2 — Spring Web (MVC)

Spring Web MVC + Embedded Tomcat spring-boot-starter-web

What it is: The web module of the Spring Framework implementing the Model-View-Controller (MVC) design pattern. It includes an embedded Apache Tomcat server so no external server installation is required.

How we use it: We define 5 controller classes — AuthController, DashboardController, FileController, FolderController, and ShareController — each annotated with @Controller. These handle all 14+ HTTP endpoints (GET/POST) for login, registration, file operations, folder management, and link sharing.

What it does in the project: Maps browser requests (e.g., GET /dashboard) to Java methods, injects form data into DTOs, populates the Model object with data for Thymeleaf templates, and returns rendered HTML pages to the browser.

4.3 — Spring Security

Spring Security spring-boot-starter-security

What it is: A powerful authentication and authorization framework for Java applications. It provides session management, password encoding, CSRF protection, and URL-based access control out of the box.

How we use it: Configured in SecurityConfig.java with a SecurityFilterChain bean. We use session-based authentication (not JWT) because it integrates seamlessly with server-side Thymeleaf rendering. Passwords are hashed with BCrypt. Public pages (/login, /register, /share/**) are whitelisted; all other URLs require authentication.

What it does in the project: Protects all private endpoints, handles the login/logout flow, hashes passwords with automatic salting, prevents CSRF attacks, manages HTTP sessions, and integrates with Thymeleaf via sec:authorize tags for conditional UI rendering (e.g., showing the username in the navbar).

4.4 — Spring Data JPA + Hibernate ORM

Spring Data JPA / Hibernate spring-boot-starter-data-jpa

What it is: Spring Data JPA is an abstraction layer over JPA (Jakarta Persistence API) and Hibernate ORM. It allows developers to interact with relational databases using Java objects instead of raw SQL. Hibernate translates Java entity classes into SQL table operations automatically.

How we use it: We define 4 JPA entity classes — User, FileEntity, Folder, and SharedLink — annotated with @Entity. Each maps to a MySQL table. We then create repository interfaces (e.g., FileRepository extends JpaRepository) that auto-generate CRUD operations. Custom queries use Spring's method-name convention (e.g., findByOriginalNameContaining).

What it does in the project: Manages all database operations — creating users, storing file metadata, building folder hierarchies with self-referencing foreign keys, and resolving share link tokens. Hibernate auto-generates the database schema from entity annotations (ddl-auto=update).

4.5 — Thymeleaf Template Engine

Thymeleaf spring-boot-starter-thymeleaf

What it is: A modern server-side Java template engine for HTML. Unlike JSP, Thymeleaf templates are valid HTML files that can be opened directly in a browser for prototyping. It uses th:* attributes to inject dynamic data.

How we use it: We have 7 HTML templates — login.html, register.html, dashboard.html, files.html, shared.html, plus reusable fragments (header.html, footer.html). Templates use th:each for looping over file lists, th:if for conditional alerts, th:text for displaying dynamic data, and th:action for secure form submissions with CSRF tokens.

What it does in the project: Renders all HTML pages on the server before sending to the browser. Displays file tables, folder grids, dashboard statistics, search results, and share link pages with live data from the controllers' Model objects.

4.6 — Jakarta Bean Validation

Hibernate Validator (Bean Validation 3.1) spring-boot-starter-validation

What it is: A declarative validation framework that uses annotations like @NotBlank, @Email, @Size on DTO fields to enforce input rules automatically.

How we use it: Applied on UserRegistrationDto to validate registration form fields — ensuring usernames are 3–50 characters, emails are properly formatted, and passwords meet minimum length requirements. Controllers use @Valid to trigger validation before processing.

What it does in the project: Prevents invalid data from reaching the database. Automatically generates error messages displayed in Thymeleaf templates when users submit invalid forms, providing immediate user feedback.

4.7 — MySQL Database

MySQL + MySQL Connector/J 8.0+

What it is: MySQL is an open-source relational database management system. MySQL Connector/J is the official JDBC driver that allows Java applications to communicate with MySQL.

How we use it: CloudNest stores all persistent data in a MySQL database called cloudnest_db with 4 tables: users, files, folders, and shared_links. The connection is configured in application.properties. The schema uses InnoDB engine with foreign key constraints and cascading deletes.

What it does in the project: Serves as the single source of truth for all application data — user accounts, file metadata, folder hierarchies, and share links. Supports complex queries for search, statistics aggregation (total files, storage usage), and node distribution analytics.

4.8 — Project Lombok

Lombok 1.18.x

What it is: A compile-time annotation processor that auto-generates boilerplate Java code — getters, setters, constructors, builders, toString(), and equals()/hashCode() — via simple annotations.

How we use it: Every entity and DTO class uses Lombok annotations: @Getter, @Setter, @NoArgsConstructor, @AllArgsConstructor, and @Builder. For example, the FileEntity class has 9 fields but zero manually written getters/setters — Lombok generates them all at compile time.

What it does in the project: Reduces entity classes from ~150 lines to ~40 lines each. Keeps the codebase clean and focused on business logic rather than boilerplate. The @Builder pattern enables fluent object construction in service classes.

4.9 — Bootstrap 5.3 + Bootstrap Icons

Bootstrap + Bootstrap Icons 5.3.3 / 1.11.3

What it is: Bootstrap is the world's most popular CSS framework for building responsive, mobile-first web interfaces. Bootstrap Icons is its companion icon library with 2,000+ SVG icons.

How we use it: Loaded via CDN in every template. We use Bootstrap's grid system for responsive layouts, its component library for modals (file upload, folder creation), alerts (success/error messages), dropdowns (user menu), tables (file listings), and buttons. Bootstrap Icons provide visual cues throughout the UI (cloud icons, folder icons, lock icons, etc.).

What it does in the project: Provides the responsive foundation for the entire frontend. Combined with our custom style.css (dark glassmorphism theme, gradient accents, animated background shapes), it creates a premium, modern UI that works seamlessly on both mobile and desktop devices.

4.10 — Apache Maven

Apache Maven 3.8+

What it is: A build automation and dependency management tool for Java projects. It uses a pom.xml file to define project structure, dependencies, and build plugins.

How we use it: The pom.xml declares all 10 dependencies (Spring Boot starters, MySQL connector, Lombok). Maven downloads them automatically from Maven Central. The Spring Boot Maven Plugin packages the application as an executable JAR for deployment.

What it does in the project: Manages the entire build lifecycle — dependency resolution, compilation, testing, and packaging. Ensures all team members use identical library versions through the Spring Boot parent POM's dependency management.
